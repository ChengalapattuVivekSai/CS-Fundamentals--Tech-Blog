# Module 12 - System Programming

## Purpose
Use operating system primitives directly and connect APIs like fork, exec, pipes, threads, and sockets to runtime behavior.

## Outcome
Write code that interacts with the OS deliberately rather than treating it as a black box.

## Linux Focus
- fork/exec/wait
- signals
- pthread primitives
- sockets
- resource leaks

## Standard Study Flow
1. Problem before this concept existed
2. Why simple approaches failed
3. Core definition and abstraction
4. Internal mechanism
5. Linux connection
6. Backend or systems relevance
7. Performance implications
8. Common bugs or misconceptions
9. Interview framing
10. Visualization and summary flow
