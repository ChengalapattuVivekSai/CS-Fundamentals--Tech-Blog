# Module 14 - OS to Networking Bridge

## Purpose
Prepare for networking from the OS side by grounding sockets, kernel buffers, and multiplexing in earlier OS concepts.

## Outcome
Enter networking with strong process, memory, syscall, and I/O context already in place.

## Linux Focus
- socket abstraction
- kernel buffers
- blocking behavior
- I/O multiplexing

- `01-socket-as-os-abstraction.md` - Socket as OS Abstraction
- `02-kernel-buffers.md` - Kernel Buffers
- `03-blocking-vs-nonblocking-sockets.md` - Blocking vs Non-Blocking Sockets
- `04-io-multiplexing.md` - I/O Multiplexing
- `05-why-networking-depends-on-os.md` - Why Networking Depends on OS

## Standard Study Flow
1. Problem before this concept existed
2. Why simple approaches failed
3. Core definition and abstraction
4. Internal mechanism
5. Linux connection
6. Backend or systems relevance
7. Performance implications
8. Common bugs or misconceptions
9. Interview framing
10. Visualization and summary flow
